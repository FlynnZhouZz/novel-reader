## 修复书架接口 500 报错

**问题**：`GET /users/me/bookshelf` 返回 `{"code":500,"message":"Bookshelf is not associated to Novel!"}`

**根因**：`novelService.ts` 的 `getMyBookshelf` 函数使用了 `Novel.findAndCountAll` 并在 `include` 中以 `model: Bookshelf, as: 'collectedBy'` 做关联查询。但 `collectedBy` 是 `Novel.belongsToMany(User, ...)` 定义的别名，对应 model 是 `User`，不是 `Bookshelf`，导致 Sequelize 报关联不存在错误。

**修复**：改为两步查询：
1. 先 `Bookshelf.findAndCountAll({ where: { user_id } })` 拿到 novelIds 和 count（分页在此处做）
2. 再 `Novel.findAll({ where: { id: novelIds } })` 带 chapters include 查小说详情

文件：`server/src/services/novelService.ts`，TypeScript 编译通过（tsc --noEmit 0 errors）
